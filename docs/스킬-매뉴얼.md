# AI Berkshire 스킬 사용 매뉴얼

> **Claude Code 가치투자 리서치 · 사용 매뉴얼**

> "Price is what you pay, value is what you get." — Warren Buffett

워런 버핏 · 찰리 멍거 · 돤융핑 · 리루 — 네 거장의 방법론을 명령어로 옮긴 투자 리서치 스킬 모음. 한 사람과 Claude가 하나의 리서치 팀이 된다.

## 먼저 알아둘 것: 네 개의 렌즈, 하나의 결론

모든 스킬은 같은 네 시각으로 기업을 본다. 단순 분업이 아니라 **서로 도전하도록** 설계되어 있다 — 한 사람이 "좋은 사업"이라 하면 다른 사람은 "어떻게 망하는가"를 묻는다.

| 렌즈 | 거장 | 무엇을 보는가 |
|------|------|--------------|
| 비즈니스 본질 | 돤융핑 | 이 사업은 본질적으로 무엇으로 돈을 버는가. 좋은 사업의 특성과 모델. |
| 경제적 해자 | 워런 버핏 | 경쟁자로부터 초과수익을 지키는 구조적 우위와 안전마진. |
| 역발상 리스크 | 찰리 멍거 | 거꾸로 생각한다. 무엇이 이 투자를 무너뜨리는가, 어떤 편향에 빠졌는가. |
| 장기 확실성 | 리루 | 10년 뒤에도 존재하는가. 문명·산업 트렌드와 경영진 리스크. |

**세 가지 기둥**

- **멀티 에이전트 병렬** — 팀형 스킬은 독립 에이전트 4~7개를 동시에 투입한다. 검색량·시각이 그만큼 배가된다.
- **데이터 엄밀성** — 핵심 수치는 2개 독립 출처로 교차검증, 오차 >1% 표시. 계산은 도구로, LLM 암산 금지.
- **강제 결론** — 양시론 금지. 통과 / 회색지대 / 제외를 가격 구간과 함께 강제 출력한다.

## 시작하기: 설치하고 호출하기

스킬 정의(`.md`)를 Claude Code의 commands 디렉터리에 복사하면, 슬래시 명령으로 바로 부른다. 명령어 이름 = 파일 이름.

```bash
# 저장소 클론
git clone https://github.com/xbtlin/ai-berkshire.git

# 스킬을 전역 commands 디렉터리에 복사
cp ai-berkshire/skills/*.md ~/.claude/commands/
```

```bash
# 한 기업을 본격 분석
/investment-team 텐센트

# 10분 빠른 선별
/investment-checklist 마오타이, 핀둬둬

# 실적 정독
/earnings-review 텐센트 2025Q4
```

> **인자 표기** — 아래 `<꺾쇠>`는 필수 입력, `[대괄호]`는 선택 입력이다.

## 리서치 여정: 언제 무엇을 쓰는가

스킬은 순서가 정해진 게 아니라 상황에 맞춰 고른다. 전형적인 흐름은 발굴에서 추적까지 다음 다섯 단계를 따른다.

**01 · 발굴 · 선별** — 후보를 추려 심층 연구할 가치가 있는지 빠르게 가린다. 한 종목이면 체크리스트, 여러 종목·지수·테마면 품질 선별, 산업 전체면 퍼널·전경 스캔.
`/investment-checklist` · `/quality-screen` · `/industry-funnel` · `/industry-research`

**02 · 심층 분석** — 통과한 기업을 본격 해부한다. 가장 빠르고 포괄적인 첫 선택은 팀형. 경영진이 핵심이면 경영진 심층, 비상장이면 탐정식, 실적 시즌이면 재무제표 정독.
`/investment-team` · `/investment-research` · `/management-deep-dive` · `/private-company-research` · `/earnings-review`

**03 · 보유 · 추적** — 매수 후의 규율 시스템. 투자 논문이 반증됐는지 분기마다 추적하고, 포트폴리오 전체의 집중도·상관관계를 점검하며, 주가가 튀면 10분 안에 원인을 가린다.
`/thesis-tracker` · `/portfolio-review` · `/news-pulse`

**04 · 공유 · 발행** — 리서치를 공개 채널 수준의 글로. 한 기업을 8편 시리즈로 완전 해부하거나, 한 주제를 바로 발행 가능한 글로 다듬는다.
`/deep-company-series` · `/wechat-article`

**05 · 상시 도구** — 여정 어디서든 부른다. 막힐 때 돤융핑식 사고로 묻고, 공급망 병목을 상시 추적하며, 모든 스킬은 공통 데이터 규범을 내장한다.
`/dyp-ask` · `/bottleneck-hunter` · `financial-data`(내장 규범)

## 스킬 레퍼런스 (16개 + 보조 2개)

각 스킬의 호출 형식 · 무엇을 하는가 · 언제 쓰는가 · 실행 흐름 · 출력 위치. **병렬**은 에이전트를 동시에 투입하는 팀형 스킬이다.

### 🔬 심층 리서치 (5)

#### `/investment-team <기업명>` · 에이전트 4 병렬
4명의 독립 분석가(비즈니스·재무·경쟁·리스크)를 동시에 투입한다. 가장 빠르고 포괄적인 첫 선택지.
- **언제** — 한 기업을 본격 분석하는 기본 출발점
- **흐름** — 4역할 분배 → 병렬 리서치 → Team Lead 종합
- **출력** — `reports/{회사}/` 4관점 + 최종보고서

#### `/investment-research <기업명>` · 단일 · 7단계
4대 거장 프레임워크를 한 컨텍스트에서 7단계로 순차 실행하는 종합 심층 리서치.
- **언제** — 깊고 일관된 단일 보고서가 필요할 때
- **흐름** — 데이터 → 본질 → 해자 → 역발상 → 경영진 → 트렌드 → 밸류에이션
- **출력** — `reports/{회사}/{회사}-research-{YYYYMMDD}.md`

#### `/management-deep-dive [인물] <회사>` · 단일
"주식을 사는 건 사람을 사는 것." 경영진의 능력·성실성·자본배분·지배구조를 검증한다.
- **언제** — 경영진이 투자의 핵심 변수일 때
- **흐름** — 능력 → 성실성 추적 → 자본배분 → 부재 시나리오
- **출력** — `reports/{회사}/{회사}-management-{YYYYMMDD}.md`

#### `/private-company-research <회사명>` · 에이전트 6 병렬
정보가 희소한 비상장사를 탐정식으로. 재무를 다중 출처로 역추정하고 신호의 일관성을 검증한다.
- **언제** — 앤트그룹·SpaceX 등 비상장 기업
- **흐름** — 6역할 병렬 → 교차검증 → 정보 사각지대 선언
- **출력** — `reports/{회사}/{회사}-private-{YYYYMMDD}.md`

#### `/deep-company-series <기업명>` · 장문 · 8편
약 12만 자 8편 시리즈로 기업을 완전 해부 — 인식 리셋부터 매도 신호까지.
- **언제** — 공개 발행급 깊이의 시리즈가 필요할 때
- **흐름** — 5년 자료 조사 → 01~08편 순차 작성 → 일관성 스캔
- **출력** — `reports/{회사}/《看懂..》/0X-*.md`

### 📊 재무제표 분석 (2)

#### `/earnings-review <회사> <분기>` · 단일
원본 재무제표만 읽고 워런 버핏처럼 분석한다. 주석에 숨은 정보와 경영진 어조까지 본다.
- **언제** — 실적 발표 직후, 2차 리포트 의존 없이
- **흐름** — 원문 수집 → 핵심 추출·검증 → MD&A 정독 → 주석 발굴
- **출력** — `reports/{회사}/{회사}-earnings-{기간}.md`

#### `/earnings-team <회사> <분기>` · 에이전트 6 병렬
4대 거장이 병렬로 실적을 해석하고, 편집자·독자 검토를 거쳐 바로 발행 가능한 글로 다듬는다.
- **언제** — 실적 해설을 공개 채널에 올릴 때
- **흐름** — 4대가 병렬 → 초안 → 편집·독자 검토 → 확정
- **출력** — `reports/{회사}/` 5역할 + 발행본

### 🏭 산업 선별 (4)

#### `/industry-research <산업/트렌드>` · 단일
공급망 전경도를 그리고 각 고리의 선도 기업을 4대 거장 방식으로 분석, 포트폴리오 배분까지 제안.
- **언제** — 한 산업의 모든 투자 기회를 훑을 때
- **흐름** — 논리 체인 → 가치사슬 전경 → 선도 기업 분석 → 배분
- **출력** — `reports/{산업}-industry-{YYYYMMDD}.md` (루트)

#### `/industry-funnel <산업/방향>` · 단일 · 퍼널
전체 시장(30~60개) → 1차 선별(≤10) → 정밀 분석 → 최종 3개사. 탈락 사유까지 남긴다.
- **언제** — 한 방향에서 소수 종목으로 정선할 때
- **흐름** — 전체 스캔 → 5지표 선별 → 정밀 → 상호보완 3선
- **출력** — `reports/{산업}-funnel-{YYYYMMDD}.md` (루트)

#### `/quality-screen <기업들|지수|테마>` · 단일 · 하드지표
7개 하드 지표로 비일류 기업을 신속 제거하되, 3가지 면제 조건으로 우량주 오판을 막는다.
- **언제** — 여러 종목·지수·테마를 빠르게 걸러낼 때
- **흐름** — 입력 파싱 → 7지표 검증 → 면제 확인 → 분류
- **출력** — 인라인 비교표 (파일 저장 없음)

#### `/investment-checklist <기업들>` · 단일 · 10분
워런 버핏 6단계 체크리스트로 심층 연구 가치를 10분 안에 판단. 마지막은 거울 테스트.
- **언제** — 본격 분석 전 빠른 1차 판단
- **흐름** — 능력범위 → 사업 → 해자 → 경영진 → 안전마진 → 규율
- **출력** — 통과/조건부/회색 비교표 + 거울 테스트

### 📈 포지션 관리 (3)

#### `/portfolio-review <보유 비중 목록>` · 단일
개별 포지션 건강도, 집중도, 상관관계, 기회비용을 종합 점검하고 리밸런싱을 제안한다.
- **언제** — "기업 리서치"에서 "포트폴리오 관리"로
- **흐름** — 보유 파싱 → 포지션 검진 → 조합 분석 → 최적화
- **출력** — `reports/portfolio-latest.md` (루트, 지속 갱신)

#### `/thesis-tracker <회사> [분기검사]` · 단일 · 장기
투자 논문을 작성하고 분기마다 핵심 가정·레드라인을 추적한다. 매도 규율 시스템.
- **언제** — 매수 후 논문 반증 여부를 지속 추적
- **흐름** — 논문 작성 → 가정 검증 → 레드라인 → 건강도 점수
- **출력** — `reports/{회사}/{회사}-thesis.md`

#### `/news-pulse <회사> [변동/기간]` · 에이전트 4 병렬
주가 급등·급락 시 10~15분 안에 4차원(기업·규제·경쟁·심리)으로 원인을 가린다.
- **언제** — 포지션이 튀어 "무슨 일이지?" 싶을 때
- **흐름** — 4차원 병렬 조사 → 성질 판단(가치/심리/불명)
- **출력** — `reports/{회사}/{회사}-news-{YYYYMMDD}.md`

### 🧠 사고 도구 · 규범 (2)

#### `/dyp-ask <질문>` · 단일 · 사고
돤융핑(大道至简)의 사고 체계와 말투로 비즈니스·투자·인생 질문에 답한다.
- **언제** — 판단이 막혔을 때, 본질로 돌아갈 때
- **흐름** — 8대 사상(능력범위·Stop doing·문화 등)으로 답
- **출력** — 직접 답변 (보고서 아님)

#### `financial-data` · 규범 · 비호출
직접 부르는 명령이 아니라 모든 스킬에 내장된 데이터 규범. 출처 우선순위와 교차검증 기준.
- **미국주** — macrotrends + stockanalysis
- **홍콩주** — aastocks + macrotrends(ADR)
- **A주** — 东方财富 + 巨潮资讯 · 오차 >1% 표시

### ➕ 보조 도구 (README 공식 16개 외, 저장소에 함께 존재)

#### `/bottleneck-hunter <슈퍼트렌드>` · 상시 스캔
3~5년 확실한 성장 트렌드의 물리적 공급망을 Layer 0~4로 분해해 "병목" 기업을 발굴한다.
- **언제** — 확실한 트렌드의 진짜 수혜 지점을 찾을 때
- **흐름** — 트렌드 검증 → 공급망 분해 → 병목 등급(S/A/B)
- **출력** — `reports/bottleneck-map/`

#### `/wechat-article <주제>` · 에이전트 다중
리서치 → 작가 초고 → 편집자·독자 검토를 거쳐 3000~4000자 발행 가능한 글을 만든다.
- **언제** — 한 주제를 대중 채널 글로 발행할 때
- **흐름** — 3 리서치 병렬 → 작가 → 편집·독자 → 확정
- **출력** — `reports/.../公众号-{주제}-{YYYYMMDD}.md`

## 공통 규칙

어떤 스킬을 부르든 모든 보고서에 동일하게 적용되는 원칙. 출력의 신뢰성은 여기서 나온다.

- **객관성 우선** — 사실과 의견을 엄격히 구분한다. 선입견 없이 데이터 → 논리 → 결론 순으로 전개하고, 모든 핵심 판단에 반대 논거를 함께 제시한다.
- **2개 출처 교차검증** — 핵심 데이터는 최소 2개 독립 출처로 대조하고 오차 >1%는 표시한다. 추정값은 `[추정]`으로 명시한다.
- **계산은 도구로** — PE·시가총액 등은 `tools/financial_rigor.py`로 정밀 계산(`Decimal`). LLM 암산 금지, 시총은 주가×주식수로 수작업 검증.
- **통화 단위 명시** — 홍콩달러·위안화·미달러를 항상 구분 표기해 단위 혼동을 막는다.
- **정보 풍부도 등급** — 보고서 첫머리에 A(충분)·B(적정)·C(희소) 등급을 매겨 "자료 많음 = 확실함" 착각을 방지한다.
- **한국어 · 출처 명시** — 모든 보고서는 한국어로, 직접적이고 간결하게. 데이터에는 출처를, 거장 어록은 요소요소에 인용한다.
