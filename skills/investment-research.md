# 투자 리서치: 워런 버핏-찰리 멍거-돤융핑-리루 4대 거장 종합 분석 프레임워크

$ARGUMENTS 에 대한 체계적인 투자 리서치 분석을 수행한다.

## 리서치 프레임워크

워런 버핏, 찰리 멍거, 돤융핑, 리루 네 명의 투자 거장 방법론을 기반으로, 다음 7개 모듈을 순서대로 실행한다.

### 사전 단계: AI 리서치 편향 자각 (필수 실행)

리서치를 시작하기 전에, 해당 기업의 "AI 리서처빌리티"를 평가하여 잠재적 데이터 편향을 식별한다.

**정보 풍부도 등급**:
| 등급 | 특징 | AI 리서치 함정 | 대응 전략 |
|------|------|-----------|---------|
| A급 (정보 충분) | 상장 다년, 증권사 커버리지 다수, 미디어 보도 밀집 | 컨센서스 과잉, AI 출력이 시장 가격에 수렴, 알파 제한 | 반면 검증에 집중: 똑똑한 사람들은 왜 사지 않는가? 간과된 리스크는? |
| B급 (정보 적정) | 상장 1~3년, 커버리지 제한, 일부 데이터 추산 필요 | AI가 "합리적 추정"으로 공백을 채울 수 있어 완전해 보이지만 실제론 거짓 확실성 | 추산 데이터마다 신뢰도 표기, "근거 있는 추산"과 "임의 채움" 구분 |
| C급 (정보 희소) | 신규 상장/비인기 종목/신흥 시장, 커버리지 거의 없음 | AI가 자료 부족으로 과도하게 보수적, "파악 불가 = 나쁨"으로 오판 | 제1원리 질문 사용(아래 참조), 제한된 정보에서 비즈니스 본질 추출 |

**C급 기업의 제1원리 리서치 방법**:
공개 자료가 부족할 때, "완전해 보이는" 보고서를 짜맞추려 하지 말고, 다음 핵심 질문에 집중한다.
1. 고객은 누구인가? 왜 돈을 내는가? 대안이 있는가?
2. 재구매는 무엇이 구동하는가? 습관, 락인, 아니면 지속적 가치 창출?
3. 경쟁자가 1000억 원을 가지고 이 사업을 복제할 수 있는가?
4. 경영진은 어떤 핵심 결정을 내렸는가? 그 결정들이 반영하는 판단력과 가치관은?

**편향 자기점검 체크리스트** (리서치 전 과정에서 경계 유지):
- [ ] 나의 "확실성" 감각은 비즈니스 본질에서 오는가, 자료 양에서 오는가?
- [ ] 이 기업의 자료량을 절반으로 줄이면 내 결론이 바뀌는가?
- [ ] AI 출력 분석이 시장 컨센서스와 과도하게 유사한가? 그렇다면 나의 정보 우위는 어디에 있는가?
- [ ] "공개 자료는 적지만 비즈니스 본질이 탁월한" 가능성이 과소평가되고 있지는 않은가?

정보 풍부도 등급 결과를 보고서 첫머리에 작성하고, 최종 결론에는 "AI 리서치 신뢰도"와 "실제 투자 확실성"의 차이를 명기한다.

### 1단계: 데이터 수집

> **데이터 소스 규범**: `skills/financial-data.md` 참조. 모든 재무 데이터는 두 개의 독립 소스에서 확인, 오차 >1%는 표기 필수.
> - 미국 주식: macrotrends (주) + stockanalysis (부)
> - 홍콩 주식: aastocks (주) + macrotrends ADR (부)
> - A주: 东方财富 (주) + 巨潮资讯 (부)

Task 도구로 백그라운드 에이전트를 실행하여 다음 데이터를 수집한다.

1. 수익 구조: 최근 회계연도 및 최근 4분기 부문별 매출, 성장률, 총이익률
2. 재무 지표: 최근 5년 매출, 순이익, 총이익률, 영업이익률, 잉여현금흐름(FCF), 현금 보유액
3. 경쟁 구도: 시장 점유율, 주요 경쟁사 비교
4. 비즈니스 모델과 경제적 해자: 핵심 경쟁 우위 원천
5. 기술 역량: 핵심 기술 스택, R&D 투자
6. 경영진: 창업자/CEO 이력, 지분율, 핵심 의사결정 기록
7. 산업 전망: TAM(총 유효 시장), 성장 예측
8. 리스크 요인: 지정학, 규제, 공급망 등
9. 현재 밸류에이션: 시가총액, PE, PS, PEG, EV/Revenue
10. 매수·공매도 측 핵심 논거

#### 데이터 교차 검증 (필수 실행, 금융 엄밀성 도구 사용)

데이터 수집 완료 후, **`tools/financial_rigor.py` 를 반드시 호출하여 핵심 데이터를 프로그램적으로 검증**한다. LLM의 암산 오류를 방지한다.

**반드시 검증할 데이터 포인트**:
- 총 발행 주식 수 (거래소, Yahoo Finance, StockAnalysis 등 최소 2개 소스 확인)
- 현재 주가 및 시가총액 (**주가×총주식수 수작업 계산 후 보고서 시가총액과 대조, 단위 오류 방지**)
- 최근 회계연도 매출 및 순이익 (기업 연간보고서 + 최소 1개 제3자 소스 확인)
- 현금 보유액 및 순현금 (현금+단기투자-총부채, 기준 차이 주의)
- 경영진 지분율 (경제적 권리와 의결권 구분, 이중 주식 구조 주의)

**강제 검증 단계 (Bash로 도구 호출)**:

Step 1 — 시가총액 검산 (정확한 십진수, 부동소수점 아님):
```bash
python3 ~/ai-berkshire/tools/financial_rigor.py verify-market-cap \
  --price {주가} --shares {총주식수} --reported {보고 시가총액} --currency {통화}
```

Step 2 — 핵심 데이터 다중 소스 교차 검증:
```bash
python3 ~/ai-berkshire/tools/financial_rigor.py cross-validate \
  --field {필드명} --values '{"소스1": 수치, "소스2": 수치}' --unit {단위}
```
매출, 순이익, 현금 보유액에 대해 각각 실행.

Step 3 — 밸류에이션 지표 정확 검산 (PE/PB/ROE/FCF Yield 등):
```bash
python3 ~/ai-berkshire/tools/financial_rigor.py verify-valuation \
  --price {주가} --eps {EPS} --bvps {주당순자산} --fcf-per-share {주당FCF} --dividend {주당배당}
```

**검증 규칙**:
1. 핵심 데이터 포인트마다 최소 2개의 독립 소스
2. 소스 간 차이 발견 시, 기업 연간보고서/거래소 데이터 우선 채택, 차이 원인 명기
3. **계산이 필요한 데이터는 모두 도구로 검산, LLM 암산 금지**
4. 도구 출력 결과는 보고서 부록 "핵심 데이터 교차 검증 기록"에 직접 삽입
5. 도구가 ❌ 편차 과다를 보고하면, 원인 규명 후에만 분석 진행

**흔한 오류 방지**:
- 시가총액 단위: 홍콩달러 억 vs 인민폐 억 vs 미국달러 억, 0 하나 쓰거나 빠뜨리기 쉬움
- FCF 기준: 소스마다 자본지출 정의가 다를 수 있음 (리스, 인수 포함 여부 등)
- 부채 기준: 운용리스 부채 포함 여부
- 지분율: 이중 주식 구조 기업의 경제적 권리 ≠ 의결권

### 2단계: 비즈니스 본질 분석 — 돤융핑 "올바른 사업"

분석 포인트:
- 이 사업의 본질을 한 문장으로 정의
- 수익 구조 분해 (차트)
- 5년 수익성 추세 (차트)
- 비즈니스 모델 캔버스: 일회성 판매 vs 구독/재구매? 하드웨어 vs 소프트웨어 vs 플랫폼?
- 생태계 점착성/고객 락인 강도
- 총이익률 수준과 동종업계 비교, 높거나 낮은 이유 설명
- 영업 레버리지 분석
- **돤융핑식 질문**: 이 사업은 어디가 좋은가? 한 문장으로만 표현한다면?

### 3단계: 경제적 해자 평가 — 워런 버핏 "경제적 해자"

다섯 가지 경제적 해자 유형을 하나씩 검증한다.

| 해자 유형 | 검증 방법 |
|-----------|---------|
| 브랜드/가격 결정력 | 매출 손실 없이 가격을 올릴 수 있는가? |
| 전환 비용 | 고객이 경쟁 제품으로 이동하는 비용은 얼마인가? |
| 네트워크 효과 | 사용자가 늘수록 제품이 더 좋아지는가? |
| 규모의 경제 | 규모로 얻는 비용 우위는 얼마나 큰가? |
| 기술/특허 장벽 | 기술 선도는 몇 년인가? 복제 가능한가? |

경제적 해자 추세 분석: 지난 5년간 넓어졌는가, 좁아졌는가? 향후 5년 전망.

**워런 버핏식 질문**: 10년 후에도 이 해자가 남아 있는가? 무엇이 그것을 파괴할 수 있는가?

### 4단계: 역발상 사고와 리스크 체크리스트 — 찰리 멍거 "거꾸로 생각하라"

- "이 기업이 실패할 수 있는 모든 경로" 나열 (표: 경로/확률/영향도)
- 역사적 유사 사례: 과거에 비슷한 위치에 있던 기업을 찾고, 그 결과는?
- 학제간 분석: 네트워크 효과 이론, 기술 채택 곡선, 경쟁 게임 이론 등으로 교차 검증
- 편향 자기점검: 서사 편향, 앵커링 효과, 생존자 편향
- 공매도 측 핵심 논거 수집

**찰리 멍거식 질문**: 내가 가장 틀릴 가능성이 높은 곳은 어디인가? 똑똑한 사람들이 왜 이 기업을 사지 않거나 공매도하는가?

### 5단계: 경영진 평가 — 돤융핑 "올바른 사람" + 워런 버핏 "경영진 성실성"

- CEO/창업자 핵심 의사결정 복기 (표: 시기/결정/결과/평점)
- 자본 배분 능력: R&D 수익률, M&A 성공률, 자사주 매입 타이밍
- 주주 이익 일치성: 경영진 지분, 보수 구조, 주식 매도 기록
- 조직 역량: 팀 안정성, 핵심 인재 리스크
- 기업 문화 특성

**돤융핑식 질문**: CEO가 은퇴해도 이 기업은 경쟁력을 유지할 수 있는가?

### 6단계: 산업과 문명적 트렌드 — 리루 "문명 진화 프레임워크"

- 해당 산업이 "문명적 패러다임 전환" 국면에 있는지 판단
- 역사적 기술 혁명 유사 사례 (증기기관/전기/인터넷/AI)
- TAM 성장 곡선과 천장 분석
- 기업의 산업 가치사슬 내 위치
- 기술 로드맵 리스크
- 고객/공급업체 집중도 분석

**리루식 질문**: 20년 후 되돌아보면, 이 기업은 "이 시대의 스탠다드 오일"인가, "반짝 사라진 3Com"인가?

### 7단계: 밸류에이션과 안전마진 — 워런 버핏 "내재가치" + 돤융핑 "올바른 가격"

- 현재 시장 가격 (핵심 밸류에이션 지표 표) — **도구로 반드시 검산**
- 역방향 DCF: 현재 주가에 내재된 성장 기대치는?
- 3가지 시나리오 밸류에이션 — **도구로 정확히 계산, 암산 금지**:
```bash
python3 ~/ai-berkshire/tools/financial_rigor.py three-scenario \
  --price {주가} --eps {EPS} --shares {총주식수억} \
  --growth {낙관성장률} {중립성장률} {비관성장률} \
  --pe {낙관PE} {중립PE} {비관PE} --years 3 --currency {통화}
```
- 자체 역사적 밸류에이션과 비교
- 동종업계 밸류에이션과 비교

**돤융핑식 질문**: 주식 시장이 내일부터 5년간 문을 닫는다면, 이 가격에 보유할 의향이 있는가?

## 출력 요건

1. 모든 분석은 데이터 근거 필수, 데이터 출처 첨부
2. 핵심 데이터는 Markdown 표로 표현
3. 각 모듈 말미에 해당 거장의 "질문" 필수
4. 완성된 보고서는 `~/[회사명]투자리서치보고서.md` 에 저장
5. 결론은 명확하게, 매수/관망/회피 권고를 회피하지 말 것
6. 밸류에이션 섹션은 구체적인 가격 범위 제시 필수
7. **보고서 첫머리**에 "정보 풍부도 등급" (A/B/C) 및 "AI 리서치 한계 선언" 포함 필수
8. **보고서 말미**에 "AI 분석 신뢰도"와 "투자 확실성"을 구분 필수 — 전자는 자료량에 의존, 후자는 비즈니스 본질에 의존. 독자에게 명확히 고지: 이 보고서의 어떤 결론이 충분한 데이터에 기반하고, 어떤 결론이 제한된 정보 추론에 기반하는지
9. 기업이 C급 (정보 희소)이면, 보고서 말미에 "직접 검증이 필요한 질문 목록" 필수 — 독자가 현장 조사, 제품 체험, 공급망 인터뷰 등을 통해 AI의 사각지대를 보완하도록 권고

## 데이터 표본검사 (출고 프로세스)

보고서 파일 저장 후, **반드시** 데이터 표본검사를 실행하고, 통과 후에만 발행한다.

**Step 1 — 표본검사 목록 추출 (15% 무작위 샘플링):**
```bash
python3 ~/ai-berkshire/tools/report_audit.py extract \
  --report <보고서 파일 경로>
```
JSON 템플릿 출력, 각 항목에 `fetched_value` (입력 대기) 포함.

**Step 2 — 데이터 검증:**
목록의 각 데이터 포인트를 `skills/financial-data.md` 규범에 따라 신뢰할 수 있는 소스에서 확인
(미국 주식: macrotrends+stockanalysis; 홍콩 주식: aastocks+macrotrends; A주: 东方财富+巨潮资讯),
`fetched_value` / `fetched_source` / `fetched_value2` / `fetched_source2` 입력.

**Step 3 — 판정 출력:**
```bash
python3 ~/ai-berkshire/tools/report_audit.py verdict \
  --results '<입력된 JSON>' \

```
