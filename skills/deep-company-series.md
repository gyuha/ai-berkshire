# 심층 기업 시리즈: 8편 장문으로 기업 하나를 완전히 해부한다

$ARGUMENTS 를 위한 8편 심층 장문 시리즈를 작성한다. 공개 채널(공개 계정/영상 채널 등)에 게재한다. **핵심 IP는 "잘 쓰는 것"이 아니라 "잘 고치는 것"——99%의 재경 기사는 이 skill의 팩트체크 기준을 위반하고 있다**.

참고 샘플: `reports/腾讯/《看懂腾讯》/`

---

## 1. 사용 시나리오

사용자가 한 기업에 대해 "교과서급" 심층 연구를 수행하고, **시리즈 장문** 형태로 공개 발행하길 원할 때 사용한다. 단편 리서치와의 차이:
- 8편 약 12만 자, 인식 리셋부터 의사결정 프레임워크까지 완전한 클로즈드 루프
- 각 편이 독립된 글(단편 공유에 적합), 단 하나의 가치평가/경영진/가격 판단 체계를 관통
- "90분을 투자해 기업 하나를 이해하려는" 독자 대상, 증권사 고객 대상이 아님

**이 skill이 적합하지 않은 경우**: 단편 리서치, 분기 실적 점평, 산업 리서치——그런 용도에는 `/investment-research`, `/earnings-review`, `/industry-research` 를 사용한다.

## 2. 시리즈 편목 템플릿 (8편)

| # | 편명 템플릿 | 핵심 질문 | 분량 |
|---|---------|---------|------|
| 01 | 당신은 X를 안다고 생각했지만, 사실 그렇지 않다 | 인식 리셋: 3가지 흔한 착각 타파 | 4,000-5,000 |
| 02 | X의 경제적 해자——`<사업 본질 한 문장>` | 해자가 깊은지, 향후 5/10년에도 유지되는지 | 6,000-8,000 |
| 03 | X의 최대 수익 엔진——`<가장 돈이 되는 사업>` | 주력 사업이 무엇인지, 왜 지속 가능한지 | 6,000-8,000 |
| 04 | X의 장부에 숨겨진 또 하나의 회사——`<숨겨진 자산>` | 투자 포트폴리오 / 자회사 / 숨겨진 가치 | 8,000-10,000 |
| 05 | AI(또는 현재 내러티브) 시대, X는 승자인가 패자인가 | 시대 변수: 사업 부문별 AI 영향 분해 | 8,000-10,000 |
| 06 | 워런 버핏 방식으로 X 재무제표를 해부한다 | 재무 심층: 매출총이익률/FCF/ROE/SBC | 8,000-10,000 |
| 07 | `<경영진 명언>`——X의 경영진에게 맡길 수 있는가 | 자본배분 규율 + 신뢰 검증 + 후계자 | 8,000-10,000 |
| 08 | 얼마면 살 만하고, 어떤 신호가 나오면 반드시 팔아야 하는가 (시리즈 최종장) | DCF 3시나리오 + 레드라인 리스트 + 포지션 프레임워크 | 10,000-12,000 |

`00-시리즈 안내.md` 를 목차 인덱스로 추가하되, 발행하지 않는다.

## 3. 작성 스타일 규범

### 어조

- **직접적·간결·군더더기 없이**——첫 문장에서 바로 수치나 반상식 결론을 제시
- **가치투자 프레임워크**——워런 버핏/찰리 멍거/돤융핑/리루 시각을 곳곳에 삽입(단, 명언 쌓기 금지)
- **입장 선입견 금지**——데이터를 먼저 제시하고, 논리를 추론하고, 마지막에 결론을 도출
- **양면 제시**——모든 핵심 판단에 "하지만 반대 측에서 보면..."을 반드시 첨부
- **공개 계정 체감**——첫 18-20자가 독립적으로 성립해야 함(모바일 미리보기)

### 금지어

| 금지 | 이유 | 대체 |
|------|------|------|
| 분명히 / 반드시 / 틀림없이 | 주관적 절대화 | 데이터에 따르면 / 증거가 보여주듯 |
| ~라고 생각한다 / ~느낌이다 | 주관적 어조 | 삭제하거나 "이 프레임워크에 따르면"으로 교체 |
| 교과서급 / 신의 한 수 | 트래픽 낚시성 칭찬 | 구체적 사실 기술 |
| 심각하게 불일치 / 심각하게 저평가 | 강한 주관적 표현 | 구체적인 할인율 % 제시 |
| 완벽 / 흠잡을 데 없다 | 일방적 판단 | 반대 측 관찰 추가 |

### 제목 스타일

- **반전 수치** 또는 **반통념 결론**을 낚시로 사용 ("15년 7차례 도전 전부 실패", "연봉 4,292만 원이 이익의 0.0017%")
- 부제목은 중립적·내용 요약 ("——`<본질 판단>`")
- **트래픽 낚시성 비유 금지**: "소형 워런 버핏", "중국판 X", "YYDS" 일체 사용 금지
- 전문 독자에게 익숙한 용어 사용 ("버크셔"이지 "워런 버핏"이 아님, 인명보다 회사명 우선)

## 4. 엄격한 팩트체크 체크리스트 (핵심 IP)

### 글 쓰기 전에 경계해야 할 "가짜 정확성" 함정

1. **확률 가중 기댓값**: `30% × A + 50% × B + 20% × C = 기대 +X%` 식의 계산은 거의 전부 쓰레기——확률 배분이 순전히 주관적이며 독자에게 잘못된 정확성 감각을 심어준다. **시나리오 + 발동 조건 + 방향만 나열하고, 가중 기댓값은 계산하지 않는다**.
2. **제3자 추정 MAU/점유율**: QuestMobile/치마이/CBNData 등 기준 차이가 매우 크다(동일 시점에서 2-3배 차이가 날 수 있음). **가장 신뢰할 수 있는 두 개를 앵커로만 활용하고, 나머지는 정성적으로 기술**.
3. **과거 성장률 선형 외삽**: `2025년 +33% × 5년 복합 → 2030년 X` 는 금융 문맹식 예측. **시나리오 가정 + 고/저 범위 + "이건 약속이 아님"**.
4. **비공개 지분율**: 바이트댄스, Halti류 비상장사 지분은 **공개 공시된 적이 없다**. **범위를 제시하고 "알 수 없음"으로 표기**.
5. **강한 귀인**: 경쟁사 실패 = X 때문. 복수의 원인을 모두 나열하고, **이 글에서는 단일 귀인을 하지 않는다**.

### 수정 시 반드시 실행할 7가지 점검

```
□ 1. 편 간 수치 일관성: 총 시가총액, Non-IFRS 순이익, 핵심 지분 % 전 시리즈 정렬
□ 2. 기준 표기: Non-IFRS / GAAP / Non-IFRS-SBC / FCF 각각 어느 기준 사용인지 전문 명확히
□ 3. 중복 합산 스캔: 이미 연결 처리된 자회사를 "투자 포트폴리오"에 중복 계상하지 않기, SOTP 이중 계산 금지
□ 4. 횡단 비교 공정성: "주력 사업 PE(현금+포트폴리오 제외)" vs "경쟁사 PE(미제외)" 방식 금지
□ 5. 확률 가중치 전부 삭제: 위 항목 참조
□ 6. 절대화 표현 전부 약화: grep "분명히|반드시|심각|교과서|완벽"
□ 7. 제3자 데이터 출처 표기: 재무제표 외 모든 데이터 뒤에 "(출처: X)" 표기
```

### 모델 오류 선호 패턴

글 쓰기 전 **이미 알려진 하드 에러 리스크를 먼저 나열**:
- 과거 수익 배수: 반드시 누적 투입 기준을 사용 (예: Riot 33배, 58배가 아님)
- 지분율: 반드시 최신 부투/재무제표 기준 확인 (예: 텐센트가 메이퇀 보유 1.5%, 6.4%가 아님)
- "배당 분배" 회계 처리: IFRIC 17에 따라 선언일 기준 처분 이익으로 인식 (예: 징둥은 2021년, 메이퇀은 2022년이지만 금액 소규모)
- 총 주식수 반등: SBC가 연초 집중 부여되면 단기적으로 주식수가 상승함

## 5. 실행 프로세스

### 1단계: 조사 (01-02편 작성 전 완료)

1. 최근 5년 연간보고서, 최신 분기 보고서 열람
2. 독립 셀사이드 리서치 최소 3편 열람 (컨센서스 + 반컨센서스 발굴)
3. `/investment-team` 또는 `/investment-research` 로 내부 연구 초안 먼저 생성
4. 사용자와 8편의 핵심 논지 확인 (다 써놓고 방향이 틀렸다는 걸 뒤늦게 발견하는 상황 방지)

### 2단계: 작성 (01→08 순서대로, 건너뛰기 금지)

- 각 편 완성 후 `reports/{회사명}/《看懂{회사명}》/0X-XX.md` 에 저장
- 즉시 GitHub 푸시 금지——사용자 검토 대기
- 사용자 수정 의견 접수 후 수정
- 수정 완료 후에만 git push

### 3단계: 편 간 일관성 스캔 (8편 전부 완성 후)

Explore 에이전트를 병렬 파견하여 8편을 대상으로 다음 점검 실시:
1. 동일 수치(시가총액, 순이익, 지분율)가 편 간 일치하는지
2. 동일 용어(FBS, SBC, Non-IFRS)가 첫 등장 시 설명되는지
3. 인용 관계: 02편이 "06편 참조"라고 했다면 실제로 대응하는지
4. 요점 정리 vs 본문 수치 일치 여부

### 4단계: 발행 전 최종 점검

```bash
# 푸시 전 반드시 로컬에서 한 번 grep (ai-berkshire 프라이버시 규칙에 따라)
grep -r "linxuan\|/Users/\|<사용자 회사 별명>" reports/ | head
```

이상 없음 확인 후에만 `git pull --rebase && git commit && git push`.

## 6. 수정 의견 처리 프로세스

사용자가 수정 의견을 줄 때, 다음 순서로 처리:

### 1. 먼저 팩트 확인 (바로 수정하지 말 것)

사용자가 "X 데이터가 틀렸다"고 하면, 먼저 Bash/Read로 원본 데이터를 교차 검증:
- ai-berkshire 프로젝트 내 동일 기업의 earnings/재무제표 보고서 확인
- 부투/공식 공시 확인
- "사용자가 제시한 데이터 vs 내가 확인한 데이터 vs 내가 기존에 사용한 데이터" 3자 비교 제시

### 2. 수정 수준 판단

| 수준 | 유형 | 처리 |
|------|------|------|
| 🔥 하드 에러 | 수치 오류, 귀인 오류, 기준 오류 | 반드시 수정, 망설임 없이 |
| ⚠️ 주관화 | 강한 주관적 표현, 절대화, 트래픽 낚시성 비유 | 약화 또는 삭제 |
| 🔬 세밀도 | 출처 표기, 기준 세분화 | 우선순위 낮음, 가독성과 균형 |
| ❓ 신뢰 불가 | 제3자 추정치 간 큰 차이 | **삭제가 수정보다 안전** (사용자 명시 지시에 따름) |

### 3. 수정 후 연동 점검

한 곳을 수정하기 전에 "이 수치/개념을 인용하는 다른 곳이 어디인가"를 먼저 생각한다. 예:
- 총 시가총액 변경 → 전 시리즈 연동하여 PE / 주력 사업 PE / 할인율 / FCF Yield 수정
- 지분 % 변경 → TOP 10 순위 + 과거 지분표 + 축소 리스트 수정
- 용어 기준 변경 → 최초 정의 + 이후 인용 + 요점 정리 수정

### 4. 푸시 후 즉시 보고

```
푸시 완료 (commit hash).
[N]건 수정 요약 [표 포함]:
- 무엇을 수정했는가
- 연동하여 무엇을 수정했는가
- 아직 수정하지 않은 것이 있는가

다음 지시 대기.
```

## 7. 이 skill이 하지 않는 것

- **독자 대신 투자 결정을 내리지 않는다**——모든 편 말미에 "투자 조언이 아님"
- **주가를 예측하지 않는다**——"시나리오 + 발동 조건"만 제시
- **"기대 연환산 수익률" 가중값을 계산하지 않는다**——주관적 확률 배분은 독자를 오도한다
- **"X 거물도 보유 중"이라고 쓰지 않는다**——타인의 포지션으로 자신의 판단을 정당화하는 것은 반가치투자적이다
- **8편 모두 작성을 강요하지 않는다**——특정 편에 충분한 독립 콘텐츠가 없다면(예: 경영진이 특별하지 않은 기업), 다른 편에 합치거나 편수를 줄인다

## 한 줄 요약

**《기업 이해 X 시리즈》의 핵심 역량 ≠ 잘 쓰는 것, 엄격하게 고치는 것**——
89%의 재경 장문은 가짜 정밀 수치, 주관적 가중 기댓값, 절대화 표현 때문에 죽는다. 이 skill의 존재 이유는 이 모든 함정을 미리 표시해 두고, 쓰기 전에 피하게 하고, 쓴 후에 깨끗이 청소하게 만드는 것이다.
