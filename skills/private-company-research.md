# 비상장 기업 리서치: 멀티 에이전트 병렬 심층 연구 프레임워크

$ARGUMENTS에 대한 팀 기반 심층 리서치 분석을 수행한다. 蚂蚁集团, 小红书, SpaceX, Stripe 등 비상장 기업을 위해 설계되었다.

**최종 목표**: 정보가 본질적으로 부족한 조건에서, 이 기업의 **진짜 가치**를 최대한 복원한다. 시장이 부여한 밸류에이션이 아닌, 사업 자체가 얼마나 가치 있는지를 판단한다.

## 프레임워크 특징

비상장 기업 vs 상장 기업 리서치의 핵심 차이:
- **표준화된 재무제표 없음**: 다중 소스에서 퍼즐 조각처럼 수집·교차 검증 필요
- **밸류에이션 앵커 부족**: 펀딩 라운드, 유사 기업 비교법, 시나리오 추론에 의존
- **정보 비대칭 심각**: '퍼즐식' 연구 방법론이 더 필요
- **엑시트 경로 불확실**: IPO/M&A/세컨더리 모두 가능

## AI 리서치 편향 자각 (이 프레임워크의 핵심 전제)

비상장 기업은 AI 리서치 편향이 가장 심각한 영역이다. 다음 함정을 항상 경계해야 한다:

**핵심 모순**: AI는 구조화된 기존 정보 처리에 강하지만, 비상장 기업의 정보는 본질적으로 부족하다. 이로 인해:
1. **가짜 보수주의**: 자료가 적으면 AI는 보수적/모호한 결론을 내리려 하지만, 자료 부족 ≠ 나쁜 기업
2. **가짜 정밀성**: 보고서 템플릿을 채우기 위해 AI가 "합리적 추측"을 "근거 있는 분석"처럼 위장할 수 있음
3. **벤치마크 함정**: 상장 기업과 강제 비교 시 상장 기업의 밸류에이션 논리를 그대로 가져오면서 비상장 기업의 고유한 가치를 놓침
4. **생존 편향**: 온라인에서 검색 가능한 기업 정보는 긍정적 편향을 가짐 (기업이 자발적으로 전파하는 뉴스는 주로 좋은 소식)

**대응 원칙**:
- 확실성을 가장한 추측으로 표를 채우기보다, "모른다"고 솔직히 남겨두는 편이 낫다
- 모든 데이터 포인트는 신뢰도를 표시해야 한다 (🟢높음/🟡중간/🔴낮음). 독자 스스로 판단하게 한다
- "검증 가능한 사실"과 "AI의 추론"을 구분하여 서로 다른 포맷으로 표시한다
- 정보가 극도로 부족한 기업은 "제1원리 모드"로 전환한다. 보고서 완결성보다 핵심 질문 몇 가지에 집중한다:
  1. 이 사업은 어떤 실제 문제를 해결하는가? 수요는 진짜인가, 가짜인가?
  2. 왜 이 팀인가? 그들의 고유한 강점은 무엇인가?
  3. 성공할 경우 천장은 얼마나 높은가? 실패할 경우 가장 가능성 높은 사인 포인트는 어디인가?
  4. 현재 단계에서 핵심 검증 마일스톤은 무엇인가?

**정보 비대칭의 역이용**: 시장이 비상장 기업에 대한 정보를 덜 보유 → 가격 결정 효율성 낮음 → 오히려 초과 수익의 원천이 될 수 있다. AI 리서치의 목표는 정보 비대칭 해소(불가능)가 아니라, 제한된 정보에서 가장 핵심적인 판단 근거를 추출하는 것이다.

```
### 2. 핵심 재무 지표 추산
아래 데이터를 최대한 추산한다. **각 데이터 포인트에 반드시 표시**: 출처, 시기, 신뢰도, 추산 방법.

- 총 수익 규모 및 성장률 (최근 3년, 연간/분기 단위)

|------|------|------|---------|---------|--------|--------|------------|------|
```

### 4단계: 6개 병렬 에이전트 시작

Agent 도구를 사용하여 동시에 6개 에이전트를 시작한다 (**같은 메시지에서 반드시 병렬 호출**):

```
당신은 {회사명} 비상장 기업 리서치 팀의 "{역할 이름}" 역할이다.

당신은 **비상장 기업**을 리서치 중이다. 이것이 의미하는 바:
- 표준화된 공개 재무제표가 없어, 여러 소스에서 정보를 수집해야 한다
- 데이터가 불완전하거나 상충할 수 있으므로, 신뢰도를 표시해야 한다
- 더 많은 추론과 합리적 추산이 필요하지만, 추산 과정을 투명하게 보여야 한다
- 정보를 찾지 못함 ≠ 정보 부재. 찾은 정보에는 편향이 있을 수 있다

아래 리서치 태스크를 완료하라: {태스크 subject}

구체적 요구사항:
{태스크 description 내용}

**리서치 방법**:
1. WebSearch를 사용하여 최신 공개 정보를 검색한다. 각 차원에서 최소 3-5회 검색, 다양한 키워드 조합 사용
2. 검색 키워드 전략:
   - 중국어: 회사명+收入/估值/融资/用户数/MAU/IPO/招股书/裁员/整改
   - 영어: Company Name + revenue/valuation/funding/users/IPO/filing
   - 구체적 인명+회사명 (경영진 관련 정보 검색)
   - 회사명+구체적 경쟁사명 (경쟁 동향 검색)
3. 주요 정보 소스:
   - 높은 신뢰도: 투자설명서, 규제 문서, 상장 기업 연차보고서 내 관련 공시
   - 중간 신뢰도: 晚点LatePost, The Information, 36氪, Bloomberg, Reuters, TechCrunch
   - 보조 검증: 知乎, 脉脉, Glassdoor, 天眼查, 企查查
4. WebFetch를 사용하여 핵심 기사 전문을 가져온다 (검색 요약만 보지 않음)
5. 중요 데이터는 최소 2개의 다른 소스로 교차 검증한다

**데이터 표시 기준 (엄격히 준수)**:
- 모든 핵심 데이터는 출처 표시 (구체적인 미디어명 및 기사 제목)
- 데이터 시기 표시 (연월까지)
- 신뢰도 표시: 🟢높음 (투자설명서/공식 공시) / 🟡중간 (신뢰할 수 있는 미디어/리서치) / 🔴낮음 (추산/소문)
- 다른 소스의 데이터가 충돌할 때, **전부 나열**하고 차이와 판단을 설명한다
- "사실"과 "추론"을 구분: 사실은 일반 텍스트, 추론/추산은 *기울임 표시*하고 추산 방법을 표시한다
- 획득 불가한 정보는 "데이터 없음"으로 명확히 표시하고, 만들지 않는다

**출력 요구사항**:
- 보고서는 상세하게, 핵심 데이터는 Markdown 테이블로 표시한다
- 각 분석 차원에는 명확한 결론과 평점이 있어야 한다
- 추산 과정은 완전히 투명하게 (계산 논리와 각 단계별 가정 포함)
- 보고서 말미에 다음을 제공한다:
  1. 이 차원의 종합 평점 (★1-5) 및 핵심 판단
  2. 이 차원 정보 완결성 자체 평가 (충분/적절/불충분/심각하게 불충분)
  3. 가장 중요한 3가지 발견
  4. 가장 큰 정보 사각지대 (어떤 정보의 부재가 판단에 가장 영향을 미치는가)
```

   - 보고서에 "긍정 정보 상세, 부정 정보 간략"의 편향이 있는지 확인한다

## 중요 주의사항

1. **6개 에이전트는 반드시 병렬 시작** — 같은 메시지에서 Agent 도구를 6번 호출한다
2. **데이터 신뢰도 표시** — 비상장 기업 데이터 소스는 품질이 다양하므로, 모든 핵심 데이터에 출처와 신뢰도를 표시한다
3. **추산 과정 투명** — 모든 추산 과정에서 계산 논리를 보여준다. 근거 없이 숫자를 제시하지 않는다
4. **교차 검증** — 핵심 데이터는 최소 2개 소스로 교차 검증하고, 소스 충돌 시 모두 나열한다
5. **신호 일관성 검증** — 종합 단계에서 반드시 차원 간 신호 일관성 검사를 수행한다
6. **결론은 명확하게** — 투자/관망/회피 추천을 피하지 않는다. 단 결론의 신뢰도를 함께 밝힌다
7. **인내심을 갖고 대기** — 6개 에이전트 리서치에는 수 분이 소요된다. 사용자에게 실시간으로 진행 상황을 업데이트한다
8. **한·영 검색** — 비상장 기업 정보는 한국어/중국어/영어 미디어에 분산될 수 있으므로, 여러 언어로 검색한다
9. **편향 방지 핵심 원칙** — 자료 부족 ≠ 나쁜 기업, AI 분석 분량 짧음 ≠ 투자 확실성 낮음. 정보가 극도로 부족한 기업은 "제1원리 모드"로 전환하여 핵심 질문에 집중하고, 보고서 형식 완결성을 추구하지 않는다
10. **솔직한 여백** — 보고서에서 "근거 있는 분석"과 "추측 채우기"를 명확히 구분하고, "이 차원의 데이터가 불충분하여 의미 있는 결론을 내릴 수 없다"는 표현이 나와도 된다
11. **대체 데이터는 노이즈가 아니다** — 채용, 특허, 소송, 앱 데이터 등 대체 데이터는 뉴스 보도보다 실제 경영 상태에 더 가까울 수 있다
12. **진짜 가치 지향** — 최종 목표는 이 사업이 얼마나 가치 있는지를 판단하는 것이지, 보기 좋은 보고서를 작성하는 것이 아니다. 정보가 부족하여 신뢰할 만한 밸류에이션 판단이 불가능하다면, 직접 "정보 불충분, 신뢰할 수 있는 밸류에이션 제시 불가"라고 말한다
